# 高风险领域提示

如果被审 skill 落在以下领域，应默认提高审慎等级，并明确说明风险来自哪里。

## 1. 浏览器会话与登录态复用

典型场景：
- 复用 Chrome/Chromium profile
- 使用 cookies、sessions、CDP 端口
- 让外部脚本接入已登录浏览器

主要风险：
- 账号会话泄露
- 越权访问已有登录状态
- 自动化脚本影响浏览器状态

默认判断：
- 若仅说明原理且不自动执行：通常 **中**
- 若直接提供会话复用脚本：通常 **高**
- 若还伴随隐蔽外联或导出会话：通常 **极高**

## 2. 认证、令牌、凭据处理

典型场景：
- OAuth / OIDC / JWT / API keys
- 读取 `.env`
- 导入 client secret / refresh token
- 自动处理 auth headers

主要风险：
- 凭据泄露
- 配置失误导致真实安全后果
- 不必要地扩大访问范围

默认判断：
- 透明工程文档：通常 **中**
- 自动抓取或导出凭据：通常 **高**
- 凭据外传或隐蔽使用：通常 **极高**

## 3. 本地文件大范围改动

典型场景：
- 整理文件
- 批量改名
- 递归移动/删除
- 重写文档树、目录结构

主要风险：
- 用户数据错位或损坏
- 高影响写操作难以完全回滚
- 误操作范围扩大

默认判断：
- 默认 dry-run 且强确认：通常 **中**
- 默认真执行且影响范围大：通常 **高**
- 含删除、覆盖且回滚弱：通常接近 **高** 或 **极高**

## 4. 安装依赖、远程抓取与执行

典型场景：
- `pip install` / `npm install` 后直接运行
- 下载脚本后立刻执行
- 拉取远程模板、二进制或工具链

主要风险：
- 供应链注入
- 执行不透明
- 容易把审计外包给外部未知内容

默认判断：
- 明确、最小化、可验证安装流程：通常 **中**
- 远程下载并执行：通常 **高**
- 多重远程执行或混淆安装流程：通常 **极高**

## 5. 持久化与系统级改动

典型场景：
- cron / systemd / launch agents
- 修改 shell profile
- 增加自启动
- 后台守护进程

主要风险：
- 长期驻留
- 用户难以觉察
- 影响系统可信边界

默认判断：
- 明确、用户同意、可逆：通常 **高**
- 隐蔽持久化或无清晰卸载：通常 **极高**

## 6. 规避检测与滥用增强

典型场景：
- stealth automation
- captcha bypass
- proxy rotation
- anti-bot evasion
- fingerprint spoofing

主要风险：
- 增强滥用能力
- 提高规避审查与风控的能力
- 超出正常自动化边界

默认判断：
- 即便没有直接窃密，也通常至少 **高**
- 若再叠加会话复用、代理、持久化，通常 **极高**

补充规则：
- 如果一个 skill 的**核心卖点**就是规避检测、绕过风控、提升隐蔽自动化能力，那么即使没有直接看到窃密代码，也应至少判为 **高**，通常偏向 **极高**。
- 这类 skill 若只基于公开介绍页做判断，通常还应建议：
  - 拉到本地做完整源码审计
  - 审后默认不要长期保留在常用技能目录
  - 如确需改造后研究，应先向用户请示

## 7. 批量外部 API / 仓库改写

典型场景：
- 批量推送 PR
- 批量改 issue / secrets / variables
- 对外部服务做写操作

主要风险：
- 大范围误写
- 权限滥用
- 业务影响快速扩散

默认判断：
- 有明确范围限制与确认：通常 **中**
- 默认批量写入外部系统：通常 **高**
